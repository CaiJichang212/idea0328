# 营养健康管家微信小程序需求文档（整合版）

## 1. 项目概述

### 1.1 项目背景
随着人们生活水平的提高，越来越多的人开始关注自身的营养健康状况。然而，大多数人缺乏专业的营养知识，难以科学地管理日常饮食。同时，超市购买的包装食品种类繁多，消费者难以快速判断其健康程度。为了帮助用户更好地了解自己的营养摄入情况，科学调整饮食结构，并快速识别包装食品的健康程度，我们计划开发一款营养健康管家微信小程序。

### 1.2 项目目标
开发一款功能完善、用户友好的微信小程序，实现以下核心目标：
1. 帮助用户记录日常饮食，监控营养摄入，提供专业的营养建议
2. 通过拍照识别包装食品配料表，快速评估食品健康程度
3. 针对特定人群（减脂、增肌、健康管理）提供个性化营养指导
4. 促进用户养成健康的饮食习惯

### 1.3 目标用户
- 关注健康饮食的普通用户
- 减脂人群：需要控制热量摄入、选择低卡食品的用户
- 增肌人群：需要高蛋白饮食、优化营养配比的用户
- 健康管理人群：有特定健康需求（如糖尿病、高血压）的用户
- 需要进行营养管理的特定群体（如孕妇、老年人等）

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 包装食品识别与健康评估
- **功能描述**：用户通过拍照识别包装食品的配料表，系统自动分析其健康程度，给出健康评分指数
- **识别方式**：
  - 拍照上传：直接拍摄包装食品图片
  - 相册选择：从相册选择已拍摄的图片
  - 条形码扫描：快速获取包装食品信息
- **识别范围**：支持超市购买的包装零食、饮料、加工食品等带有配料表的包装食物
- **健康评估**：
  - 健康评分：根据配料表成分分析，计算并给出0-10分的健康评分指数
  - 颜色编码：绿色（优秀）、黄色（良好）、橙色（一般）、红色（较差）
  - 评分依据：配料表成分、营养成分比例、添加剂种类、加工程度（NOVA分类）
  - 营养分析：热量、蛋白质、脂肪、碳水化合物、纤维素等详细分析

#### 2.1.2 食物记录
- **功能描述**：用户可以记录每天吃的所有食物，包括一日三餐、零食、饮料、水果等
- **操作流程**：用户进入记录页面，选择食物类型，输入食物名称、重量/数量，系统自动匹配食物数据库并计算营养成分
- **详细功能**：
  - 支持手动输入食物名称
  - 支持扫码识别食物包装获取营养信息
  - 支持拍照记录食物，通过图像识别自动匹配食物信息
  - 支持快速选择常用食物
  - 支持批量添加食物
  - 支持修改和删除已记录的食物

#### 2.1.3 热量监控
- **功能描述**：统计用户吃的食物的热量，不足或超标时提醒用户
- **计算逻辑**：根据用户的体重、身高、性别、年龄、工作强度等因素，计算出用户的每日热量需求
- **详细功能**：
  - 首次使用时引导用户填写个人基本信息
  - 自动计算基础代谢率（BMR）和每日总消耗热量（TDEE）
  - 实时显示当日热量摄入与目标的对比
  - 当热量摄入不足或超标时发送提醒
  - 支持设置个性化热量目标

#### 2.1.4 营养监控
- **功能描述**：统计用户吃的食物的基础营养成分，包括蛋白质、脂肪、碳水化合物、维生素、矿物质、水、膳食纤维等，不足或超标时提醒用户
- **详细功能**：
  - 实时显示各类营养成分的摄入情况
  - 根据用户个人信息计算各类营养成分的推荐摄入量
  - 当某类营养成分不足或超标时发送提醒
  - 提供营养成分摄入比例分析

#### 2.1.5 食品添加剂监控
- **功能描述**：统计用户吃的食物中是否包含食品添加剂，不足或超标时提醒用户，例如：盐、糖、油等
- **详细功能**：
  - 识别并统计常见食品添加剂的摄入量
  - 设置添加剂摄入上限
  - 当添加剂摄入超标时发送提醒
  - 提供添加剂摄入分析报告

#### 2.1.6 针对特定人群的专属功能

##### 减脂人群专属功能
- **热量预算管理**：根据用户身体数据和减脂目标，设定每日热量预算，扫描食品时自动计算剩余预算
- **减脂友好度评分**：专门针对减脂人群，评估食品的减脂友好程度
- **低卡替代推荐**：扫描高热量食品时，自动推荐低卡、低脂的替代产品
- **饱腹感指数**：评估食品的饱腹感，帮助减脂人群选择更耐饿的食物
- **餐前决策助手**：扫描后给出"建议食用"、"适量食用"、"避免食用"的明确建议

##### 增肌人群专属功能
- **蛋白质密度评分**：评估食品的蛋白质含量和性价比
- **增肌营养配比**：分析食品的蛋白质、碳水化合物、脂肪比例，是否符合增肌需求
- **训练后推荐**：根据训练时间，推荐适合训练后食用的食品
- **蛋白补充建议**：扫描后给出蛋白质补充建议和推荐食品

##### 健康管理人群功能
- **糖尿病友好度**：评估食品的血糖指数和糖分含量，适合糖尿病人群
- **高血压人群建议**：分析钠含量，给出高血压人群的食用建议
- **过敏原提醒**：识别常见过敏原（坚果、乳制品、麸质等），提供过敏风险提示
- **特殊饮食支持**：支持生酮、素食、无麸质等特殊饮食模式

#### 2.1.7 AI营养师功能
- **功能描述**：内置AI营养师，用户可随时咨询营养相关问题
- **详细功能**：
  - 智能问答：支持文字和语音输入，解答营养相关问题
  - 个性化建议：根据用户的健康目标、身体状况、饮食偏好，提供个性化建议
  - 饮食规划：根据扫描历史，生成个性化的饮食规划
  - 营养缺口分析：分析用户营养摄入情况，指出营养缺口并给出补充建议

#### 2.1.8 营养建议
- **功能描述**：根据用户的营养需求，提供专业的营养建议，例如：增加蛋白质、减少脂肪、增加碳水化合物等
- **详细功能**：
  - 根据用户的营养摄入情况生成个性化建议
  - 提供食物推荐，帮助用户调整饮食结构
  - 支持用户设置特定营养目标（如增肌、减肥等）
  - 提供专业的营养知识科普

#### 2.1.9 营养日报、周报
- **功能描述**：生成用户的营养日报、周报，包括：热量、基础营养成分、食品添加剂、营养建议等
- **详细功能**：
  - 自动生成每日营养报告
  - 每周生成营养周报告
  - 支持查看历史报告
  - 支持报告分享功能
  - 提供营养趋势分析

### 2.2 辅助功能

#### 2.2.1 用户管理
- 用户注册与登录
- 个人信息管理（身高、体重、年龄、性别、工作强度等）

- 健康目标选择与管理

#### 2.2.2 食物数据库
- 内置常见食物营养数据库
- 支持用户自定义食物添加
- 支持食物营养信息更新
- 包装食品营养数据库，支持快速查询和分析

#### 2.2.3 数据同步
- 支持微信云同步，多设备数据共享
- 支持数据导出功能

#### 2.2.4 历史记录与收藏
- 保存用户的包装食物识别历史，方便用户查看过往记录
- 收藏功能：用户可收藏常用包装食物，方便快速查看其健康信息
- 健康趋势分析：以图表形式展示用户的饮食健康趋势变化

#### 2.2.5 社交与激励功能
- 健康打卡：记录用户选择健康食品的次数，形成打卡记录
- 成就系统：设置健康饮食成就，激励用户养成健康饮食习惯
- 社区分享：用户可分享健康食品发现，参与健康饮食讨论
- 好友对比：支持与好友对比健康饮食评分，增加趣味性

## 3. 非功能需求

### 3.1 性能需求
- 页面加载时间不超过3秒
- 数据处理响应时间不超过1秒
- 支持同时在线用户数不少于10000人
- 包装食物配料表识别准确率达到85%以上

### 3.2 可用性需求
- 界面友好，操作简单直观
- 支持离线使用，网络恢复后自动同步数据
- 提供清晰的错误提示和操作引导
- AI营养师回答准确、有用，用户满意度达到80%以上

### 3.3 安全性需求
- 保护用户个人信息和健康数据
- 数据传输加密
- 遵循微信小程序安全规范
- 用户数据和识别结果进行加密存储
- 隐私政策：明确告知用户数据使用范围和保护措施
- 用户授权：获取用户相机和相册权限前明确告知并获得授权
- 数据匿名化：对用于分析的用户数据进行匿名化处理

### 3.4 兼容性需求
- 支持微信小程序最新版本
- 适配不同尺寸的移动设备屏幕

## 4. 数据需求

### 4.1 用户数据
- 基本信息：姓名、性别、年龄、身高、体重、联系方式
- 健康信息：工作强度、健康状况、特殊需求
- 目标设置：热量目标、营养目标、健康目标（减脂、增肌、健康管理）

### 4.2 食物数据
- 食物基本信息：名称、分类、图片
- 营养成分：热量、蛋白质、脂肪、碳水化合物、维生素、矿物质、膳食纤维等
- 添加剂信息：盐、糖、油等含量
- 包装食品数据：配料表信息、营养成分表、健康评分

### 4.3 记录数据
- 饮食记录：时间、食物名称、数量、营养成分
- 识别记录：包装食品识别历史、健康评分记录
- 营养分析：每日、每周营养摄入情况
- 提醒记录：各类营养不足或超标提醒

- **识别页**：拍照识别、条形码扫描、识别结果展示（健康评分、配料表分析、营养成分详情、个性化建议、替代产品推荐）
- **记录页**：食物选择、添加、编辑功能
- **分析页**：营养成分分析、趋势图表、健康趋势分析
- **AI营养师页**：聊天界面，支持文字和语音输入
- **建议页**：个性化营养建议、食物推荐
- **我的页**：个人信息、设置、历史记录、收藏管理、成就展示

### 5.3 交互设计
- 手势操作：支持滑动、点击、长按等操作
- 动画效果：平滑过渡、加载动画
- 反馈机制：操作成功/失败提示、进度反馈
- 颜色编码：使用直观的颜色编码快速传达健康程度

## 11. 附录

### 11.1 术语定义
- BMR：基础代谢率，指人体在清醒而又极端安静的状态下，不受肌肉活动、环境温度、食物及精神紧张等影响时的能量代谢率
- TDEE：每日总消耗热量，指人体一天内消耗的总热量，包括基础代谢、运动消耗和食物热效应
- 营养成分：食物中含有的对人体有益的物质，如蛋白质、脂肪、碳水化合物、维生素、矿物质等
- 食品添加剂：为改善食品品质和色、香、味，以及为防腐、保鲜和加工工艺的需要而加入食品中的人工合成或者天然物质
- NOVA分类系统：根据食品加工程度对食品进行分类的系统
